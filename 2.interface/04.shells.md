The **Shells** section gives you browser-based terminal sessions on the engine host — a full xterm.js terminal connected over a WebSocket to a shell process running on the machine where the engine runs.

## The Shells Explorer

Shells live in the **Shells** section of the left explorer sidebar.

- **Header**: a refresh button re-lists your shells.
- **Empty state**: when there are none, the section shows "No shells yet" / "Sessions opened or resumed here" with a **New shell** button.
- **New shell**: available from the empty-state button, the section's right-click menu, and the **Shell** menu in the menu bar.
- **Shell node**: shows a terminal icon and the shell's identity. Clicking it opens (or resumes) the session in a tab. A node's right-click menu offers **Delete**.

## Creating a Shell

**New shell** opens a session immediately — there are no options to fill in. The engine launches the host's default shell:

| Host OS | Shell |
|---------|-------|
| Linux | `/bin/bash` |
| macOS | `/bin/zsh` |
| Windows | `C:\Windows\System32\cmd.exe` |

The process runs on the **host** (it is not a container shell), inherits the engine's environment plus `TERM=xterm-256color`, and uses the engine process's working directory.

## The Terminal

Sessions use an [xterm.js](https://xtermjs.org) terminal with:

- **Automatic resize** — the terminal refits to the pane and the new size is sent to the host PTY
- **Light/Dark theme** following the console theme, and font scaling with the Interface Scale
- **Live reconnect** — reopening a shell reattaches to the running session and resumes its live output

## Sessions

- A shell's state is one of `created`, `executed`, `failed`, `killed`, `exited`.
- **Delete** removes a single shell (from the node menu).
- **Prune all** (from the **Shell** menu) removes shells.

::callout{color="amber"}
**Not part of the shell feature**: you can't choose a shell type (bash/zsh), working directory, environment variables, or a target container from the console; there are no container or database shells, session sharing, split views, color-scheme picker, or scrollback configuration. Each session is the host's default shell.
::

See the [Shell CLI reference](/docs/cli/shell) and [Shell API](/docs/api/shell) for scripting and additional options (the API/CLI accept an optional `path` and `args` when creating a shell).
